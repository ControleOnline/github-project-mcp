# Task Completion Criteria

## Overview

Use esta skill para padronizar quando uma tarefa pode ser considerada concluida.

## Workflow

1. nao considere uma tarefa concluida so porque houve uma acao executada
2. antes de encerrar, confirme que o alvo correto foi identificado
3. confirme que as checagens aplicaveis ao caso foram executadas
4. confirme que as acoes corretivas necessarias e seguras foram realizadas, ou explicitamente descartadas com justificativa
5. confirme que o resultado foi registrado no controle ou historico operacional adequado
6. deixe claras as pendencias, riscos residuais e a proxima execucao ou validacao necessaria, quando houver
7. se qualquer item essencial ainda estiver faltando, trate a tarefa como incompleta

## Output Contract

Ao concluir, informe objetivamente:

- se a tarefa esta concluida ou nao
- o que foi confirmado para permitir o encerramento
- o que ainda esta pendente, quando houver
- qual risco residual ou proxima acao ainda existe, quando aplicavel

## Quality Bar

- nao encerre tarefas por impressao subjetiva
- nao trate checagem parcial como conclusao
- nao omita pendencias ou riscos residuais
- sempre deixe rastreavel por que a tarefa foi considerada concluida
